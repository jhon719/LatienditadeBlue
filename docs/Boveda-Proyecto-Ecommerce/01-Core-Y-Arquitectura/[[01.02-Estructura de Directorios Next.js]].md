# Estructura de Directorios Next.js (App Router)

Esta nota detalla a profundidad cómo se organiza la estructura de carpetas y archivos en el proyecto utilizando **Next.js (App Router)**. Esta organización separa de forma limpia la experiencia pública de la tienda (`(shop)`) del panel de control administrativo y POS (`(admin-panel)`), optimizando la mantenibilidad y el flujo de desarrollo para Antigravity.

---


## 🔍 Explicación Detallada de Capas y Observaciones Clave

### 1. Grupos de Rutas con Paréntesis `(shop)` y `(admin-panel)`

* **¿Por qué se usan?:** En Next.js App Router, las carpetas envueltas en paréntesis como `(shop)` y `(admin-panel)` actúan como **grupos de rutas virtuales**. Esto significa que no afectan la URL final en el navegador (por ejemplo, `/admin/dashboard` sigue siendo limpia), pero permiten aislar layouts completamente diferentes:
* El layout de `(shop)` cargará la barra de navegación de la tienda, el carrito y el pie de página estilo Homidori.
* El layout de `(admin-panel)` cargará barras laterales de control de inventario, protección estricta por roles de usuario (solo administradores) y el módulo POS.



### 2. Capa de API Routes (`src/app/api/`)

* Funciona como el backend integrado de Next.js mediante Server Actions o Route Handlers.
* Contiene la lógica crítica de seguridad contra ataques, manejo de condiciones de carrera (doble clic en pagos), recepción de webhooks de Mercado Pago de forma asíncrona, y la manipulación segura de archivos subidos por el cliente (como las capturas de POS).

### 3. Capa de Estado y Lógica (`src/stores/` y `src/lib/`)

* **`stores/`**: Mantiene la reactividad del cliente sin recargar la página (por ejemplo, actualizar el contador del carrito o abrir el modal de previsualización rápida).
* **`lib/`**: Centraliza las conexiones externas. Aquí reside la seguridad de la base de datos a través de `db.ts` (Prisma) y los adaptadores de almacenamiento en la nube.
