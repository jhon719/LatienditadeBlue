

# Visión General y Stack Tecnológico

## 1. Concepto y Modelo de Negocio

El proyecto consiste en una plataforma de e-commerce especializada en la venta de figuras coleccionables y mercancía de anime (inspirada estructural y visualmente en tiendas de referencia como **Homidori**). El sistema opera bajo un modelo híbrido que resuelve las problemáticas comunes de fraude, desconfianza y falta de trazabilidad en comercios digitales mediante:

* **Catálogo visual inmersivo** enfocado en preventas y stock directo.
* **Sistema de pagos dual:** Pasarela automatizada (Mercado Pago) y método manual asistido por código QR digital y validación de comprobantes (*vouchers*) en un entorno POS interno.
* **Trazabilidad jurídica y de confianza:** Emisión automática de boletas de venta virtuales en PDF con códigos de proceso únicos (`processCode`).
* **Seguridad y comunidad:** Sistema de perfiles con datos protegidos para administradores, sistema de reseñas de usuarios y blindaje estricto contra ataques web o dobles transacciones.

## 2. Stack Tecnológico

### A. Frontend (Interfaz de Usuario)

* **Framework:** **Next.js (App Router)**. Permite renderizado híbrido (SSR/SSG), optimización automática de imágenes y una separación limpia entre las rutas de la tienda pública y el panel administrativo.
* **Estilos y UI:** **Tailwind CSS** para un diseño responsivo, limpio y minimalista. Se integran librerías de componentes y animaciones fluidas para los elementos interactivos y botones de la interfaz.
* **Gestión de Estado Global:** **Zustand** para controlar el estado del carrito de compras, modales y filtros temporales de forma ligera y rápida.

### B. Backend y Base de Datos

* **ORM:** **Prisma ORM**. Funciona como la capa de abstracción para la base de datos, garantizando consultas tipadas, migraciones controladas y protección nativa contra inyecciones SQL.
* **Base de Datos:** **PostgreSQL**. Base de datos relacional robusta encargada de almacenar de forma atómica y segura los datos de usuarios, productos, órdenes, reseñas y comprobantes de pago.

### C. Servicios Externos y Utilidades

* **Pasarela de Pagos:** SDK oficial de **Mercado Pago** para cobros automatizados con tarjetas y monederos digitales.
* **Almacenamiento de Archivos (Cloud Storage):** Proveedor en la nube (como Supabase Storage o Vercel Blob) para almacenar las imágenes de los productos, avatares de usuario y las capturas de comprobantes de pago manual.
* **Generación de Documentos:** Librería orientada a Node.js para la creación dinámica de documentos PDF (boletas de venta virtuales).

## 3. Arquitectura General del Flujo de Trabajo

```text
[ Cliente / Usuario ] 
       │
       ├──> Navegación en Tienda (Estilo Homidori) & Galería de Productos
       ├──> Selección de Compra (Stock / Preventa)
       └──> Checkout Dual:
                ├──> Opción A: Pasarela Automática (Mercado Pago) -> Webhook -> Aprobación Instantánea
                └──> Opción B: Pago Manual con QR Digital -> Subida de Voucher -> Bandeja POS (Admin)
                                                                                       │
                                                                           [ Aprobación por Admin ]
                                                                                       │
                                                                           [ Generación de Boleta PDF ]

```
