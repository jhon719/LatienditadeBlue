Este es el punto final de la cadena de valor: convertir una venta validada en un paquete físico que llega a manos del cliente. Al automatizar la generación de etiquetas, eliminamos la transcripción manual de direcciones (fuente principal de errores en envíos).

# [[06.03-Logística de Envíos e Integración con Courier]]

## 1. Visión General

El objetivo de este módulo es generar automáticamente los documentos necesarios para el despacho en **Shalom** y **Olva Courier**, basándose en los datos del pedido ya aprobado. El sistema debe preparar la información lista para "copiar y pegar" o, idealmente, generar un PDF con la etiqueta de envío.

## 2. Flujo Operativo de Despacho

1. **Activación de Despacho:** Tras la aprobación del pago en la **Bandeja POS (05.02)**, el pedido pasa al estado `READY_TO_SHIP`.
2. **Generación de Etiqueta (Packing Slip):**
* El administrador entra al pedido y presiona **"Generar Guía de Envío"**.
* El sistema genera un archivo PDF que contiene:
* **Datos del Remitente:** Tus datos precargados.
* **Datos del Destinatario:** Nombre, DNI, Teléfono y Dirección/Agencia extraídos directamente del CRM.
* **Código de Orden:** Un código de barras o QR para tu control interno.
* **Detalle de Contenido:** Lista de figuras para que el almacenero sepa qué empacar.




3. **Registro de Tracking:** Un campo editable donde el administrador pega el código de seguimiento (ej. número de guía de Shalom o tracking de Olva) para que el cliente pueda rastrear su paquete desde su perfil.

## 3. Integración con Couriers (Estrategia de Datos)

Dado que las APIs de courier nacionales suelen ser privadas o requerir contratos corporativos, el sistema utiliza **Integración Híbrida**:

* **Olva Courier:** Integración vía API (si cuentas con contrato) para generar el número de tracking directamente. Si no, se genera un formato de etiqueta estandarizado con los datos del destinatario para que solo tengas que imprimir y pegar.
* **Shalom:** Generación de un formato de "Hoja de Envío" que organiza los datos en el orden exacto que requiere el formulario manual de Shalom, permitiendo un despacho veloz.
* **Notificación Automática:** En cuanto ingresas el número de tracking en el sistema, este dispara un email/WhatsApp (manual) al cliente: *"¡Tu pedido ya está en camino! Tu número de guía es: [Número]"*.

## 4. Diseño del Etiquetado (Packing Slip)

El diseño debe ser funcional para el almacén y profesional para el cliente:

* **Formato:** A6 (tamaño ideal para etiquetas adhesivas o media hoja A4).
* **Zona de Datos:** Tipografía grande para los datos del cliente (evita errores de lectura).
* **Checklist de Packing:** Una pequeña lista con casillas de verificación para que quien empaque marque que la figura está protegida (ej: "Burbujas de aire ✅", "Caja rígida ✅").
