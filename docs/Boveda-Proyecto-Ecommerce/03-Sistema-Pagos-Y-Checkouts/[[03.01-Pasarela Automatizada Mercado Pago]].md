# [[03.01-Pasarela Automatizada Mercado Pago]]

Esta nota documenta la integración técnica, el flujo de transacciones, el manejo de estados de las órdenes y la configuración de webhooks para la pasarela de pagos oficial de **La Tiendita de Blue** utilizando **Mercado Pago**. El sistema automatiza la verificación del pago para actualizar las preventas y el stock al instante.

---

## 1. Arquitectura y Flujo de Pago E2E

El flujo está diseñado para mantener al cliente informado y asegurar que los servidores de Neon actualicen el stock local inmediatamente después de procesarse la transacción.

```text
[ Carrito / Checkout ] ──> Genera Orden en DB (Estado: PENDING)
                                   │
                                   ▼
[ SDK Mercado Pago ] ──> Crea Preference ID (Checkout Pro)
                                   │
                                   ▼
[ Pantalla de Pago ] ──> Cliente paga (Tarjeta, Efectivo, Transferencia)
                                   │
                                   ▼
[ Notificación Webhook ] ──> API Route `/api/webhooks/mercadopago`
                                   │
                                   ▼
[ Procesamiento Backend ] ──> Valida firma y cambia Estado (PAID / FAILED)

```

## 3. Flujo de Control en Base de Datos (Prisma SQL)

El modelo de órdenes se sincroniza con los estados de Mercado Pago para mapear transacciones exitosas, pendientes de efectivo o fallidas.

```prisma
enum OrderStatus {
  PENDING
  PAID
  FAILED
  CANCELLED
  DELIVERED
}

model Order {
  id            String      @id @default(uuid())
  userId        String
  user          User        @relation(fields: [userId], references: [id])
  total         Float
  status        OrderStatus @default(PENDING)
  mpPreferenceId String?    @unique // ID generado por Mercado Pago
  mpPaymentId   String?     @unique // ID del pago final procesado
  
  orderItems    OrderItem[]
  createdAt     DateTime    @default(now())
  updatedAt     DateTime    @updatedAt
}

```

## 4. Implementación del Backend (Next.js App Router)

### A. Creación de la Preferencia de Pago (`/api/checkout/route.ts`)

Cuando el cliente hace clic en "Proceder al Pago", este Server Action o API Route genera el entorno seguro de cobro con los artículos del carrito.

```ts
import { NextResponse } from "next/server";
import { MercadoPagoConfig, Preference } from "mercadopago";

const client = new MercadoPagoConfig({ accessToken: process.env.MERCADO_PAGO_ACCESS_TOKEN! });

export async function POST(req: Request) {
  try {
    const { orderId, items, userEmail } = await req.json();

    const preference = new Preference(client);
    
    // Mapear los productos al formato requerido por Mercado Pago
    const mpItems = items.map((item: any) => ({
      id: item.id,
      title: item.name,
      quantity: item.quantity,
      unit_price: item.price,
      currency_id: "PEN", // Soles Peruanos
    }));

    const response = await preference.create({
      body: {
        items: mpItems,
        payer: { email: userEmail },
        external_reference: orderId, // Vinculación crucial con nuestra DB
        back_urls: {
          success: `${process.env.NEXT_PUBLIC_BASE_URL}/checkout/success`,
          failure: `${process.env.NEXT_PUBLIC_BASE_URL}/checkout/failure`,
          pending: `${process.env.NEXT_PUBLIC_BASE_URL}/checkout/pending`,
        },
        auto_return: "approved",
        notification_url: `${process.env.NEXT_PUBLIC_BASE_URL}/api/webhooks/mercadopago`,
      },
    });

    return NextResponse.json({ preferenceId: response.id });
  } catch (error) {
    console.error("Error creando preferencia de Mercado Pago:", error);
    return NextResponse.json({ error: "Error al procesar el pago" }, { status: 500 });
  }
}

```

### B. El Webhook de Escucha Automatizada (`/api/webhooks/mercadopago/route.ts`)

Esta ruta recibe de forma asíncrona las notificaciones de Mercado Pago cuando un cliente completa su pago (incluso si cierra la pestaña de la tienda).

```ts
import { NextResponse } from "next/server";
import { MercadoPagoConfig, Payment } from "mercadopago";
import { prisma } from "@/lib/prisma"; // Instancia global de Prisma

const client = new MercadoPagoConfig({ accessToken: process.env.MERCADO_PAGO_ACCESS_TOKEN! });

export async function POST(req: Request) {
  const url = new URL(req.url);
  const type = url.searchParams.get("type") || url.searchParams.get("topic");

  // Escuchar únicamente eventos de pago terminados
  if (type === "payment") {
    const dataId = url.searchParams.get("data.id") || url.searchParams.get("id");
    
    if (!dataId) return NextResponse.json({ error: "Missing ID" }, { status: 400 });

    try {
      const payment = await new Payment(client).get({ id: dataId });
      const orderId = payment.external_reference; // Recuperamos el ID de nuestra orden
      
      if (!orderId) return NextResponse.json({ error: "External Reference not found" }, { status: 400 });

      // Si el pago fue aprobado, actualizamos la base de datos
      if (payment.status === "approved") {
        await prisma.order.update({
          where: { id: orderId },
          data: { 
            status: "PAID",
            mpPaymentId: dataId
          },
        });
        
        // Aquí se dispara opcionalmente la lógica de reducción de stock físico o confirmación de preventa
      } else if (payment.status === "rejected") {
        await prisma.order.update({
          where: { id: orderId },
          data: { status: "FAILED" },
        });
      }

      return NextResponse.json({ received: true }, { status: 200 });
    } catch (error) {
      console.error("Error en Webhook de Mercado Pago:", error);
      return NextResponse.json({ error: "Webhook internal error" }, { status: 500 });
    }
  }

  return NextResponse.json({ received: true }, { status: 200 });
}

```
