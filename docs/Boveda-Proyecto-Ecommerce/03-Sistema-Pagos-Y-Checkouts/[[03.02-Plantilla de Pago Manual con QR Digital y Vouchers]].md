Esta sección es vital para el mercado peruano, ya que las transferencias directas, **Yape** y **Plin** representan una gran parte de las conversiones.

Dado que el stack utiliza **Cloudinary** para la carga de imágenes por parte de los usuarios (como definimos en las fotos de *unboxing* de las reseñas) y servimos los recursos gráficos estáticos desde la carpeta local, esta arquitectura híbrida es ideal para gestionar los vouchers sin sobrecargar tu servidor.

Aquí tienes la redacción de la nota técnica estructurada.

# [[03.02-Plantilla de Pago Manual con QR Digital y Vouchers]]

Esta nota documenta el flujo de pagos manuales (Yape, Plin, Transferencia Bancaria BCP/Interbank) en `ecommerce-basictech`. Permite a los usuarios realizar el pago fuera de la plataforma y adjuntar una captura de pantalla (voucher) que será verificada manualmente por el administrador para liberar el stock y procesar el envío.

---

## 1. Arquitectura y Flujo de Aprobación

A diferencia de la automatización por webhooks de Mercado Pago, el pago manual requiere intervención humana para conciliar el ingreso del dinero.

```text
[ Checkout ] ──> Selecciona "Pago Manual (Yape / Plin / Transferencia)"
                      │
                      ▼
[ Renderizado de QRs ] ──> Muestra imágenes locales (Yape/Plin) y N° de Cuentas
                      │
                      ▼
[ Subida de Voucher ] ──> El cliente sube la captura de pantalla (Vía Cloudinary)
                      │
                      ▼
[ Creación de Orden ] ──> Estado en DB: AWAITING_VERIFICATION (Esperando Revisión)
                      │
                      ▼
[ Panel de Admin ] ──> El admin visualiza el voucher, verifica en su app bancaria y aprueba (Cambia a PAID)

```

---

## 2. Gestión de Archivos y Recursos

### A. Recursos Estáticos Locales (Los QRs de Cobro)

Los códigos QR oficiales de la tienda se almacenan en la carpeta local pública, ya que no cambian con frecuencia y deben cargar instantáneamente.

```text
public/
└── imagenes/
    └── pagos/
        ├── qr-yape.webp
        ├── qr-plin.webp
        └── icon-bcp.webp

```

### B. Almacenamiento Dinámico (Vouchers de Clientes)

Las capturas de pantalla de los clientes se suben a **Cloudinary** para aprovechar la optimización automática y no saturar el almacenamiento de Neon ni el disco del servidor de Next.js.

---

## 3. Adaptación del Modelo de Datos (Prisma Schema)

El modelo de órdenes (`Order`) requiere campos adicionales para identificar el método de pago elegido y almacenar la URL de la imagen del comprobante subido a Cloudinary.

```prisma
enum OrderStatus {
  PENDING
  AWAITING_VERIFICATION // 👈 Nuevo estado: Esperando validación del voucher
  PAID
  FAILED
  CANCELLED
  DELIVERED
}

enum PaymentMethod {
  MERCADO_PAGO
  MANUAL_TRANSFER // 👈 Para Yape, Plin o Cuenta Bancaria
}

model Order {
  id              String        @id @default(uuid())
  userId          String
  user            User          @relation(fields: [userId], references: [id])
  total           Float
  
  status          OrderStatus   @default(PENDING)
  paymentMethod   PaymentMethod @default(MERCADO_PAGO)
  
  // Datos específicos del Pago Manual
  voucherUrl      String?       // URL de Cloudinary de la captura de pantalla
  
  // Datos de Mercado Pago (Opcionales si elige Manual)
  mpPreferenceId  String?       @unique
  mpPaymentId     String?       @unique
  
  orderItems      OrderItem[]
  createdAt       DateTime      @default(now())
  updatedAt       DateTime      @updatedAt
}

```

## 4. UI del Frontend: Componente de Pago Manual (`<ManualPaymentView/>`)

Este componente de React (usando Tailwind CSS y Radix UI) se renderiza en el paso final del checkout si el usuario selecciona el método manual.

```tsx
"use client";

import { useState } from "react";
import Image from "next/image";
import { UploadCloud, CheckCircle } from "lucide-react";
// Asumiendo que usas next-cloudinary o un widget personalizado
import { CldUploadWidget } from "next-cloudinary"; 

export default function ManualPaymentView({ onConfirmOrder }: { onConfirmOrder: (voucherUrl: string) => void }) {
  const [voucherUrl, setVoucherUrl] = useState<string | null>(null);

  return (
    <div className="bg-white p-6 rounded-2xl border border-gray-100 shadow-sm space-y-6">
      <h3 className="font-montserrat font-bold text-lg text-azul-oscuro text-center">
        Pago por Transferencia / Billetera Digital
      </h3>
      
      <p className="text-sm text-gray-500 text-center">
        Escanea el código QR desde tu aplicación o transfiere a los números de cuenta indicados.
      </p>

      {/* Contenedor de QRs (Imágenes Locales) */}
      <div className="flex flex-col sm:flex-row justify-center gap-6 py-4">
        <div className="text-center space-y-2">
          <div className="w-32 h-32 relative mx-auto rounded-xl overflow-hidden border-2 border-[#742284]">
            <Image src="/imagenes/pagos/qr-yape.webp" alt="QR Yape" fill className="object-cover" />
          </div>
          <span className="font-montserrat font-bold text-[#742284]">Yape: 999 888 777</span>
        </div>

        <div className="text-center space-y-2">
          <div className="w-32 h-32 relative mx-auto rounded-xl overflow-hidden border-2 border-[#00B4D8]">
            <Image src="/imagenes/pagos/qr-plin.webp" alt="QR Plin" fill className="object-cover" />
          </div>
          <span className="font-montserrat font-bold text-[#00B4D8]">Plin: 999 888 777</span>
        </div>
      </div>

      <div className="bg-gray-50 p-4 rounded-xl text-sm text-gray-600 text-center border border-gray-100">
        <p className="font-bold mb-1">BCP Soles: 191-00000000-0-00</p>
        <p>CCI: 002-191-000000000000-55</p>
        <p className="text-xs mt-2 italic text-gray-400">Titular: La Tiendita de Blue S.A.C.</p>
      </div>

      {/* Zona de Subida de Voucher a Cloudinary */}
      <div className="border-t border-gray-100 pt-6">
        <h4 className="font-montserrat font-bold text-sm text-azul-oscuro mb-3">
          Sube tu comprobante de pago
        </h4>

          <CldUploadWidget 
            uploadPreset="tiendita_vouchers" // Configurado en tu dashboard de Cloudinary
            onSuccess={(result: any) => {

            }}
```

## 5. Panel de Administración (Gestión de Vouchers)

Para el backend operativo, el módulo de **Radix UI** en la ruta privada `/admin/orders` filtrará inicialmente todas las órdenes con el estado `AWAITING_VERIFICATION`.

El administrador interactúa de la siguiente manera:

1. Abre el detalle de la orden.
2. Hace clic en la miniatura de la imagen (URL de Cloudinary) para visualizar el voucher del cliente en tamaño completo.
3. Verifica en la app bancaria correspondiente que el monto haya ingresado.
4. Presiona el botón de acción **"Aprobar Pago"**, el cual ejecuta un Server Action en Next.js modificando el estado en Prisma de `AWAITING_VERIFICATION` a `PAID`, y desencadena la reducción del stock físico y el envío de la boleta electrónica.
