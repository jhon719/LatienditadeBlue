# [[03.03-Tipos de Envios y Coordinación por WhatsApp]]

Esta nota detalla la lógica de negocio, el modelado en base de datos y la experiencia de usuario para la gestión de entregas en **La Tiendita de Blue**. Debido a la naturaleza del mercado peruano, el sistema combina tarifas fijas de transporte con una canalización directa hacia **WhatsApp** para coordinar ubicaciones exactas, rangos horarios y compartir comprobantes de seguimiento (Olva Courier, Shalom o Motorizado).

---

## 1. Tipos de Envío y Reglas de Negocio

El sistema expone tres modalidades principales de entrega durante el checkout. Cada una altera el costo total de la orden y el flujo de empaquetado:

| Tipo de Envío | Cobertura | Costo Adicional | Dinámica de Coordinación |
| --- | --- | --- | --- |
| **Recojo en Tienda** | Punto físico designado | S/ 0.00 (Gratis) | Se le envía la dirección exacta y horario de atención por WhatsApp. |
| **Envío Local (Motorizado)** | Lima Metropolitana | Tarifa plana (Ej. S/ 10.00) | Se solicita la ubicación por GPS (Google Maps) para el repartidor. |
| **Envío a Provincias** | Todo el Perú (Olva/Shalom) | Tarifa calculada / Pago en destino | Se solicitan datos de DNI, Agencia destino y se envía foto de la guía. |

---

## 2. Modificaciones en el Modelo de Datos (Prisma SQL)

El esquema de Prisma mapea las direcciones de entrega y almacena los datos necesarios para que el administrador gestione el despacho (*fulfillment*).

```prisma
enum ShippingType {
  PICKUP          // Recojo en tienda
  LOCAL_DELIVERY  // Motorizado Lima
  NATIONAL_COURIER // Olva / Shalom a Provincias
}

enum ShippingStatus {
  PREPARING       // Empaquetando el coleccionable
  SHIPPED         // En camino / Dejado en agencia
  DELIVERED       // Entregado al cliente
}

model Order {
  id              String         @id @default(uuid())
  userId          String
  user            User           @relation(fields: [userId], references: [id])
  total           Float          // Incluye el precio de productos + costo de envío
  status          OrderStatus    @default(PENDING)
  
  // Bloque de Despacho
  shippingType    ShippingType   @default(LOCAL_DELIVERY)
  shippingStatus  ShippingStatus @default(PREPARING)
  shippingCost    Float          @default(0.00)
  
  // Dirección y Contacto
  receiverName    String
  receiverPhone   String
  deliveryAddress String?        // Opcional si es PICKUP
  trackingNumber  String?        // Código de seguimiento de Olva/Shalom o enlace del motorizado
  
  createdAt       DateTime       @default(now())
  updatedAt       DateTime       @updatedAt
}

```

## 3. Flujo de Coordinación Automatizada por WhatsApp

Para evitar fricciones y automatizar la atención sin perder el trato personalizado, el sistema genera un **Enlace Dinámico de WhatsApp** en la pantalla de confirmación de compra (`/checkout/success`).

### Construcción de la URL Dinámica

El backend o frontend procesa los datos de la orden recién pagada (o en estado de verificación si fue manual) y estructura una plantilla de texto codificada para la API de WhatsApp (`[https://wa.me/](https://wa.me/)`):

```text
https://wa.me/51997763962?text=¡Hola!+Acabo+de+realizar+un+pedido+en+La+Tiendita+de+Blue.+Mi+código+de+orden+es+*ORDER_ID*.+Elegí+la+modalidad+de+*TIPO_ENVIO*.+Quedo+atento+para+coordinar+los+detalles+de+la+entrega.

```

## 4. UI del Frontend: Componente de Confirmación (`<OrderSuccessView/>`)

Este componente (desarrollado con Tailwind CSS) felicita al usuario tras cerrar el flujo de pago e incentiva el clic inmediato de contacto para asegurar su entrega.

```tsx
"use client";

import { MessageSquare, MapPin, Truck, Store } from "lucide-react";

interface OrderSuccessProps {
  orderId: string;
  customerName: string;
  shippingType: "PICKUP" | "LOCAL_DELIVERY" | "NATIONAL_COURIER";
  totalAmount: number;
}

export default function OrderSuccessView({ order: OrderSuccessProps }) {
  const adminPhoneNumber = "51997763962"; // Número oficial de soporte de la tienda
  
  // Formatear el string del tipo de envío para el mensaje de texto
  const shippingLabels = {
    PICKUP: "Recojo en Tienda",
    LOCAL_DELIVERY: "Envío Express en Lima",
    NATIONAL_COURIER: "Envío a Provincia (Olva/Shalom)",
  };

  const message = `¡Hola! Acabo de realizar una compra en La Tiendita de Blue. Mi código de orden es *${order.orderId}*. Elegí la modalidad de *${shippingLabels[order.shippingType]}*. Quedo atento para coordinar los detalles del envío. ✨`;
  const whatsappUrl = `https://wa.me/${adminPhoneNumber}?text=${encodeURIComponent(message)}`;

  return (
    <div className="max-w-xl mx-auto my-12 p-8 bg-white rounded-3xl border border-gray-100 shadow-md text-center space-y-6">
      <div className="w-16 h-16 bg-green-50 rounded-full flex items-center justify-center mx-auto text-green-500">
        <MessageSquare size={32} className="fill-green-500/10" />
      </div>

      <div className="space-y-2">
        <h2 className="font-montserrat font-extrabold text-2xl text-azul-oscuro">
          ¡Gracias por tu compra, {order.customerName}! 🎉
        </h2>
        <p className="text-sm text-gray-500 px-4">
          Tu pedido <span className="font-mono bg-gray-100 px-1.5 py-0.5 rounded text-xs text-gray-700 font-bold">{order.orderId}</span> ha sido registrado correctamente.
        </p>
      </div>

      {/* Detalle del Tipo de Envío Seleccionado */}
      <div className="flex items-center gap-4 p-4 bg-azul-pastel/10 rounded-2xl border border-azul-pastel/20 text-left">
        <div className="p-3 bg-white rounded-xl text-azul-oscuro shadow-sm">
          {order.shippingType === "PICKUP" && <Store size={20} />}
          {order.shippingType === "LOCAL_DELIVERY" && <Truck size={20} />}
          {order.shippingType === "NATIONAL_COURIER" && <MapPin size={20} />}
        </div>
        <div>
          <h4 className="font-montserrat font-bold text-xs text-azul-oscuro uppercase tracking-wider">
            Modalidad Elegida
          </h4>
          <p className="text-sm font-semibold text-gray-700">
            {shippingLabels[order.shippingType]}
          </p>
        </div>
        <span className="ml-auto font-montserrat font-bold text-azul-oscuro">
          S/ {order.totalAmount.toFixed(2)}
        </span>
      </div>

      {/* Bloque de Acción Principal: WhatsApp */}
      <div className="bg-amber-50/50 border border-amber-100 p-5 rounded-2xl text-sm text-gray-600 space-y-4">
        <p className="leading-relaxed">
          Para proceder con el empaque y coordinar el horario de entrega o el número de seguimiento de tu encomienda, dale clic al botón de abajo.
        </p>
        
        <a
          href={whatsappUrl}
          target="_blank"
          rel="noopener noreferrer"
          className="inline-flex items-center justify-center gap-2 w-full py-3.5 bg-[#25D366] hover:bg-[#20ba5a] text-white font-montserrat font-bold rounded-xl shadow-md transition-colors text-sm tracking-wide"
        >
          <MessageSquare size={18} className="fill-white" />
          Coordinar Envío por WhatsApp
        </a>
      </div>
    </div>
  );
}

```

## 5. Panel de Administración y Despacho (*Fulfillment*)

Desde la ruta privada `/admin/orders`, el administrador visualiza el tipo de envío de cada orden pendiente y actualiza el flujo operativo:

1. **Filtro de Preparación:** El administrador empaqueta los productos guiándose por los datos de `receiverName` y `receiverPhone`.
2. **Inyección de Tracking:** Al dejar el paquete en la agencia (Olva/Shalom) o despacharlo con el motorizado, el administrador edita la orden en el panel e introduce el código de seguimiento en el campo `trackingNumber`.
3. **Notificación Final:** Al cambiar el estado de `shippingStatus` a `SHIPPED`, el sistema dispara un correo automático al cliente e introduce el número de tracking dentro del panel del perfil del usuario en la web `latienditadeblue.org.pe`.
