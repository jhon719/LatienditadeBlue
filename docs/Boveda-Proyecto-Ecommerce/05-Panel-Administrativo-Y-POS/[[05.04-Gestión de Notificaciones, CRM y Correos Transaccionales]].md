# [[05.04-Gestión de Notificaciones, CRM y Correos Transaccionales]]

## 1. Visión General

Este módulo es el "cerebro comunicativo" del sistema. Se encarga de enviar notificaciones automáticas en los momentos críticos del flujo de compra para que el cliente nunca se sienta desatendido.

## 2. El Ciclo de Notificaciones (Eventos)

El sistema debe tener plantillas pre-configuradas que se disparan según el `estado` del pedido:

* **Pedido Recibido (Pending):** Envío automático del enlace de pago manual (Proforma).
* **Pago Aprobado (Processing):** Envío de la boleta electrónica PDF y confirmación de recepción de fondos.
* **En Camino (Shipped):** Notificación con el número de seguimiento (tracking) del courier (Shalom/Olva).
* **Entrega Confirmada (Delivered):** Correo de seguimiento para asegurar que el producto llegó en buenas condiciones y pedir una reseña.
* **Alerta de Preventa (Stock Arrival):** Notificación automática a quienes reservaron un producto cuando este llega finalmente a almacén.

## 3. CRM: Ficha de Cliente (Perfil 360°)

Más allá de enviar correos, este módulo centraliza el historial del usuario:

* **Historial de Compras:** Total invertido por el cliente.
* **Nivel de Fidelidad:** Etiquetado manual o automático (ej. "Cliente VIP", "Coleccionista frecuente", "Cliente en riesgo").
* **Notas del Admin:** Campo de texto libre para que el admin deje recordatorios (ej. "Cliente prefiere que le escriban por WhatsApp antes de llamar").

## 4. Gestión de Listas (Opt-in Marketing)

* **Base de datos de suscritos:** Lista de correos de clientes que aceptaron recibir promociones (cumpliendo la Ley de Protección de Datos).
* **Seguimiento de Aperturas:** Estadísticas básicas (¿Cuántos clientes abrieron el correo de la preventa? ¿Cuántos hicieron clic?).
