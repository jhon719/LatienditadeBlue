

# [[05.03-Control de Inventario y Preventas]]

## 1. Gestión del Catálogo y Variantes (La Entidad)

Una figura no siempre llega en las mismas condiciones. Este submódulo permite que un solo producto maestro tenga diferentes versiones, evitando que llenes tu tienda de productos duplicados.

* **Producto Maestro:** Datos base como Título (Ej: *Zoro Roronoa - King of Artist*), Franquicia, Fabricante (Bandai, Banpresto) y Galería de fotos oficiales.
* **Gestión de Variantes (Condición):** Dentro del producto, puedes crear "Sub-ítems" con precios y stocks independientes:
* *Variante A:* Nuevo / Sellado (Stock: 2 | Precio: S/ 120)
* *Variante B:* Loose / Sin Caja (Stock: 1 | Precio: S/ 90)
* *Variante C:* Caja Dañada (Stock: 1 | Precio: S/ 105)


* **Trazabilidad SKU:** Generación de un código único interno para cada variante, facilitando la búsqueda rápida si en un futuro implementas un lector de código de barras.

## 2. Visor Dual de Stock (Control Físico vs. Tránsito)

Para evitar sobrevender figuras o confundir al cliente público, el panel administrativo manejará dos columnas de inventario claramente diferenciadas:

* **Stock Inmediato (Físico):** Figuras que tienes listas para empacar y enviar hoy mismo mediante Shalom u Olva. Este número se muestra públicamente en la tienda.
* **Stock en Tránsito (Lotes Aduanas):** Figuras que compraste a tu proveedor y vienen en camino. Este stock es **invisible para la venta directa**, pero es el que alimenta el sistema de Preventas.
* *Inyección Automática:* Cuando marcas en tu panel que un Lote Internacional ha sido "Recibido", el sistema mueve automáticamente las figuras del *Stock en Tránsito* al *Stock Inmediato*, publicándolas en la web al instante.

## 3. El Motor de Preventas (Flujo de Apartados)

Este es el núcleo que reemplazará tus cálculos manuales y mantendrá tus cuentas claras. Controla todo el ciclo de vida de una figura apartada:

* **Creación de la Preventa:** Se selecciona el cliente (buscándolo en la base de datos), se elige la figura en tránsito y se registra el **Monto de Adelanto** (Deposit). El sistema calcula automáticamente el **Saldo Restante**.
* **Línea de Tiempo Visual (ETA):** Cada preventa está atada a una "Fecha Estimada de Llegada". Si el barco se retrasa, actualizas la fecha del lote y todas las preventas asociadas cambian automáticamente, notificando si es necesario.
* **Gestión de Cobranzas (Semáforo):**
* 🟡 **Notificación de Llegada:** Cuando la figura llega a tu almacén, el sistema te sugiere enviar un mensaje: *"Tu figura ya llegó, tienes X días para cancelar el saldo"*.
* 🔴 **Alerta de Vencimiento:** Si el cliente no paga en el tiempo estipulado, la fila se pone en rojo.


* **Resolución:**
* *Éxito:* El cliente paga el saldo, se genera la orden de envío y la preventa se cierra.
* *Abandono:* Un botón de "Cancelar Preventa" que anula el trato, retiene el adelanto (según tus políticas) y libera la figura directamente al *Stock Inmediato* de la tienda para que otro la compre.



## 4. Alertas Automáticas (Low Stock)

* Un panel lateral que te avise cuando una figura popular se está quedando sin stock (Ej: "Quedan 2 unidades de *Goku Super Hero*"), permitiéndote tomar decisiones rápidas sobre si debes pedir más unidades a tu proveedor en tu próxima importación.
