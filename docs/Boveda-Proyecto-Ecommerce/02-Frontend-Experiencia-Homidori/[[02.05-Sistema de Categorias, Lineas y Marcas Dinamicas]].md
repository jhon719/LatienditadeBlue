
# [[02.05-Sistema de Categorias, Lineas y Marcas Dinamicas]]

Esta nota detalla la arquitectura de datos, la lógica de negocio y los componentes front-end encargados de gestionar las agrupaciones del catálogo de **La Tiendita de Blue**. El sistema permite clasificar los productos por **Categorías (Animes y Series)**, **Líneas de Figuras (Formatos/Colecciones)** y **Marcas (Fabricantes)** de manera 100% dinámica desde la base de datos, sirviendo los recursos gráficos desde el almacenamiento local.

---

## 1. Mapeo Físico y Convención de Nombres (Recursos Locales)

Para garantizar un rendimiento óptimo y eliminar dependencias externas, todas las imágenes asociadas a las clasificaciones se sirven localmente a través de la carpeta `public` utilizando el **`slug`** del registro como nombre de archivo.

```text
public/
└── imagenes/
    ├── Mascota BLUE.png
    ├── LOGO BLUE.jpg
    ├── Lista de Animes/      <-- Mapeado para Categorías (Series)
    │   ├── dragon-ball.webp
    │   ├── frieren.webp
    │   └── jujutsu-kaisen.webp
    └── Lineas de figuras/    <-- Mapeado para Líneas de Figuras
        ├── banpresto.webp
        ├── ichiban-kuji.webp
        └── nendoroid.webp

```

### Reglas de Conversión:

* **Formato Estándar:** `.webp` obligatorio para optimizar la transferencia de datos.
* **Generación de Slugs:** Los nombres se limpian eliminando caracteres especiales, tildes y espacios, sustituyéndolos por guiones (ej. *"Jujutsu Kaisen"* $\rightarrow$ `jujutsu-kaisen`).

## 2. Modelado de Datos Relacional (Prisma Schema)

El motor de base de datos en Neon almacena los flags de control que alteran el comportamiento y el orden en el Front-End.

```prisma
model Category {
  id             String    @id @default(uuid())
  name           String    @unique // Ej. "Jujutsu Kaisen"
  slug           String    @unique // Ej. "jujutsu-kaisen"
  
  // Toggles de Estado y Control Administrativo
  isActive       Boolean   @default(true)   // Oculta/Muestra en la tienda
  isTrending     Boolean   @default(false)  // Envía a la sección "Tendencias! 🔥" al inicio
  hasNewArrivals Boolean   @default(false)  // Activa destello de stock nuevo (Opcional)

  products       Product[]
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt
}

model Line {
  id             String    @id @default(uuid())
  name           String    @unique // Ej. "Ichiban Kuji"
  slug           String    @unique // Ej. "ichiban-kuji"
  
  isActive       Boolean   @default(true)
  isFeatured     Boolean   @default(false)  // Destacado en filtros secundarios
  
  products       Product[]
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt
}

```

## 3. Implementación del Frontend: Componente `<CategoryTrends/>`

Este componente es el encargado de pintar la fila horizontal scrollable en la **Home**. Aplica un algoritmo de ordenamiento para asegurar que los elementos marcados como tendencia aparezcan en primera posición e incluye el texto reflectante amarillo neón.

```tsx
"use client";

import Link from "next/link";
import Image from "next/image";

interface CategoryItem {
  id: string;
  name: string;
  slug: string;
  isTrending: boolean;
  hasNewArrivals: boolean;
  isActive: boolean;
}

export default function CategoryTrends({ categories }: { categories: CategoryItem[] }) {
  // 1. Filtrar solo las categorías que están activas para el público
  const activeCategories = categories.filter(cat => cat.isActive);

  // 2. Ordenar de forma descendente por el flag de tendencia (true va primero)
  const sortedCategories = [...activeCategories].sort(
    (a, b) => Number(b.isTrending) - Number(a.isTrending)
  );

  return (
    <section className="py-6 px-4">
      {/* Título de la Sección en la Portada */}
      <h3 className="font-montserrat font-extrabold text-xl text-azul-oscuro flex items-center gap-2 mb-4">
        Tendencias! 🔥
      </h3>

      {/* Contenedor con Scroll Horizontal oculto */}
      <div className="flex gap-4 overflow-x-auto pb-4 scrollbar-none snap-x">
        {sortedCategories.map((cat) => {
          // Mapeo directo hacia tu carpeta física "Lista de Animes"
          const localSrc = `/imagenes/Lista de Animes/${cat.slug}.webp`;

          return (
            <Link
              key={cat.id}
              href={`/catalogo?category=${cat.slug}`}
              className="group relative flex-shrink-0 w-64 h-32 rounded-[1.5rem] overflow-hidden bg-gray-100 border border-gray-100/50 shadow-sm snap-start"
            >
              {/* Texto Superior Flotante: Amarillo Neón */}
              {cat.isTrending && (
                <span className="absolute top-3 left-3 z-10 bg-[#CCFF00] text-black font-montserrat font-black text-[9px] tracking-widest px-2.5 py-1 rounded-full uppercase shadow-md animate-pulse">
                  TENDENCIA
                </span>
              )}

              {/* Imagen del Anime / Serie desde la Carpeta Local */}
              <Image
                src={localSrc}
                alt={cat.name}
                fill
                sizes="256px"
                className="object-cover group-hover:scale-105 transition-transform duration-300"
                // Fallback automático por si el archivo local aún no se sube
                onError={(e) => {
                  const target = e.target as HTMLImageElement;
                  target.srcset = "";
                  target.src = "/imagenes/Mascota BLUE.png"; // Usa la mascota como respaldo
                }}
              />

              {/* Degradado oscuro inferior para contrarrestar la tipografía blanca */}
              <div className="absolute inset-0 bg-gradient-to-t from-black/70 via-black/10 to-transparent" />

              {/* Nombre de la Categoría */}
              <span className="absolute bottom-3 left-4 text-white font-montserrat font-bold text-sm tracking-wide group-hover:text-[#CCFF00] transition-colors">
                {cat.name}
              </span>
            </Link>
          );
        })}
      </div>
    </section>
  );
}

```

## 4. Flujo de Trabajo en el Panel de Administración

Para que el sistema se mantenga cohesionado y no rompa la interfaz, el CRUD administrativo de categorías opera bajo la siguiente lógica estricta:

1. **Creación de Registro:** Al rellenar el formulario de nueva categoría con *"Chainsaw Man"*, el sistema genera dinámicamente la propiedad `slug: "chainsaw-man"`.
2. **Asignación de Estado:** El administrador puede prender o apagar los switches independientes:
* **`isActive`**: Si está apagado, se excluye de las consultas del catálogo.
* **`isTrending`**: Si está encendido, el backend le da prioridad posicional en la carga del inicio.


3. **Control Interno de Medios:** El panel incluye un cuadro de diálogo informativo que notifica al administrador:
> 📂 **Aviso de Recurso Local:** Para completar la visualización visual de este registro, asegúrate de guardar un archivo en formato WebP con el nombre `chainsaw-man.webp` dentro del directorio local: `public/imagenes/Lista de Animes/`.
