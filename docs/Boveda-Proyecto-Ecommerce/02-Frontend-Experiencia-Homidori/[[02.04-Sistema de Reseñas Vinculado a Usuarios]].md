
# [[02.04-Sistema de Reseñas Vinculado a Usuarios]]

Esta nota detalla el diseño técnico, la lógica de validación y los componentes de la interfaz para el sistema de calificaciones y reseñas de productos. Su objetivo es aportar prueba social (*social proof*) e incentivar la compra mediante opiniones reales de clientes, utilizando almacenamiento de avatares 100% local.

---

## 1. Flujo de Interacción y Reglas de Negocio

Para asegurar la autenticidad de las opiniones y evitar el spam de reseñas falsas, el sistema implementa las siguientes directrices:

```text
[ Usuario compra un Producto ] ──> [ Estado de Orden: ENTREGADO ]
                                           │
                                           ▼
[ Habilitación del Formulario en Detalle de Producto ]
                                           │
                                           ▼
[ Formulario de Reseña (Comentario + Estrellas + Fotos Unboxing) ]
                                           │
                                           ▼
[ Renderizado en la Ficha del Producto con Avatar desde Carpeta Local ]

```

### Reglas Críticas:

1. **Comprador Verificado:** Solo los usuarios autenticados que posean una orden de compra completada (`DELIVERED`) para ese producto específico pueden redactar una reseña.
2. **Límite por Producto:** Un usuario solo puede dejar una única reseña por producto adquirido. Si desea corregirla, el panel de su perfil le permitirá editar su comentario existente.
3. **Mapeo de Avatares Locales:** Cuando un usuario se registra o edita su perfil, su imagen se almacena físicamente en el servidor local. Las fotos de *unboxing* del producto adjuntas en la reseña se siguen delegando a Cloudinary para no saturar el almacenamiento del proyecto.

---

## 2. Estructura de Carpetas Locales e Integración con Base de Datos

Siguiendo la jerarquía física de recursos del proyecto, los avatares se gestionan dentro de la carpeta pública de Next.js:

```text
mi-proyecto-nextjs/
├── public/
│   ├── imagenes/
│   │   ├── Mascota BLUE.png
│   │   ├── LOGO BLUE.jpg
│   │   ├── Lineas de figuras/
│   │   ├── Lista de Animes/
│   │   └── avatars/         <-- Carpeta local donde se guardan las fotos de perfil de los usuarios

```

### Modelo de Datos (Prisma SQL)

Para enlazar las imágenes locales, el modelo `User` expone un campo `avatarFileName` que coincide con el ID o username del usuario para apuntar al archivo correcto en disco.

```prisma
model Review {
  id          String   @id @default(uuid())
  rating      Int      @default(5) // Rango del 1 al 5
  comment     String   @db.VarChar(500)
  images      String[] // URLs de Cloudinary con fotos del cliente (unboxing del producto)
  
  // Relaciones
  productId   String
  product     Product  @relation(fields: [productId], references: [id], onDelete: Cascade)
  
  userId      String
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@unique([productId, userId]) // Garantiza una sola reseña por usuario por producto
}

```

## 3. Arquitectura del Front-End (Componentes React)

### A. El Componente de Avatar Local (`<UserAvatar/>`)

Este componente renderiza la imagen del usuario almacenada en tu carpeta de avatares. Si el usuario no ha subido una foto personalizada, el sistema jala de forma automática la imagen de la mascota **Bluet** como un tierno fallback por defecto.

```tsx
"use client";

import Image from "next/image";

interface UserAvatarProps {
  avatarFileName?: string | null;
  username: string;
  size?: number;
}

export default function UserAvatar({ avatarFileName, username, size = 40 }: UserAvatarProps) {
  // Ruta dinámica apuntando a tu carpeta física local
  const srcPath = avatarFileName 
    ? `/imagenes/avatars/${avatarFileName}` 
    : "/imagenes/Mascota BLUE.png"; // Fallback oficial si no tiene foto

  return (
    <div 
      style={{ width: size, height: size }}
      className="relative rounded-full overflow-hidden border border-gray-200 bg-azul-pastel/20 shadow-inner flex-shrink-0"
    >
      <Image
        src={srcPath}
        alt={`Avatar de ${username}`}
        fill
        sizes={`${size}px`}
        className="object-cover"
        onError={(e) => {
          // Si por alguna razón el archivo local falla o se borró, carga la mascota
          const target = e.target as HTMLImageElement;
          target.srcset = "";
          target.src = "/imagenes/Mascota BLUE.png";
        }}
      />
    </div>
  );
}

```

### B. Listado de Opiniones (`<ProductReviewsList/>`)

Este componente se encarga de pintar la lista de reseñas en la interfaz de la ficha técnica del producto.

```tsx
"use client";

import { Star } from "lucide-react";
import UserAvatar from "./UserAvatar";

interface ReviewItem {
  id: string;
  rating: number;
  comment: string;
  images: string[];
  createdAt: string;
  user: {
    username: string;
    avatarFileName: string | null;
  };
}

export default function ProductReviewsList({ reviews }: { reviews: ReviewItem[] }) {
  if (reviews.length === 0) {
    return (
      <p className="font-montserrat text-sm text-gray-400 italic py-4">
        Aún no hay reseñas para este producto. ¡Sé el primero en compartir tu opinión!
      </p>
    );
  }

  return (
    <div className="space-y-6 mt-6">
      {reviews.map((review) => (
        <div key={review.id} className="border-b border-gray-100 pb-6">
          <div className="flex items-center gap-3 mb-2">
            
            {/* Renderizado desde la carpeta de imágenes locales */}
            <UserAvatar 
              username={review.user.username} 
              avatarFileName={review.user.avatarFileName} 
              size={42} 
            />
            
            <div>
              <h4 className="font-montserrat font-bold text-xs text-azul-oscuro">
                @{review.user.username}
              </h4>
              <div className="flex items-center gap-1 mt-0.5">
                {[...Array(5)].map((_, i) => (
                  <Star
                    key={i}
                    size={12}
                    className={i < review.rating ? "fill-yellow-400 text-yellow-400" : "text-gray-200"}
                  />
                ))}
              </div>
            </div>
            
            <span className="ml-auto font-montserrat text-[10px] text-gray-400">
              {new Date(review.createdAt).toLocaleDateString()}
            </span>
          </div>

          <p className="font-montserrat text-sm text-gray-600 pl-14 leading-relaxed">
            {review.comment}
          </p>

          {/* Fotos de Unboxing del cliente (Vía Cloudinary) */}
          {review.images.length > 0 && (
            <div className="flex gap-2 mt-3 pl-14">
              {review.images.map((imgUrl, index) => (
                <a key={index} href={imgUrl} target="_blank" rel="noreferrer" className="relative w-16 h-16 rounded-xl overflow-hidden bg-gray-50 border hover:opacity-90 transition-opacity">
                  <img src={imgUrl} alt="Unboxing del producto" className="w-full h-full object-cover" />
                </a>
              ))}
            </div>
          )}
        </div>
      ))}
    </div>
  );
}

```

## 4. Guardado Físico de Archivos (API Backend)

Cuando un usuario sube o cambia su avatar desde su panel de perfil, el backend procesa el archivo mediante un API Route (`/api/user/upload-avatar`) que escribe directamente en la carpeta local utilizando el módulo nativo de Node.js `fs` (File System):

1. **Destino de Escritura:** `public/imagenes/avatars/[userId].webp` (se formatea a webp para mitigar peso).
2. **Registro SQL:** En el modelo del usuario se guarda el string `"[userId].webp"` en la columna `avatarFileName`.
